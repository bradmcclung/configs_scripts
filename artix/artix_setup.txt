=== Base Installation ============================================================================================================
==================================================================================================================================

After booting the Artix ISO from USB, use the following login info:

    username: root
    password: artix

------ Wifi ------------------------------------------------------------------------------

In the terminal, run

    ip a

to check the available internet connections.
For wifi, check if an interface like wlan0 has an ip address (listed as ipnet).
If it doesn't, execute

    ip link set wlan0 up

If an error occurs that cites something like RF-kill, execute

    rfkill unblock wifi

and then

    ip link set wlan0 up

To connect to a wireless network, execute

    connmanctl

This will initiate a connman session.
Next, scan for wifi connections by executing

    scan wifi

    services

Look for your wifi connection, and take note of the first three numbers after wifi_<alphanumeric>_.
Then execute

    agent on

    connect wifi_<alphanumeric>_<first three nums>

using tab completion after wifi_ and <first three nums>.
Afterwards, enter the wifi password.
Finally, execute

    quit

to finish.
Execute

    ip a

to check that wlan0 has an ip address.

------ Partitions ------------------------------------------------------------------------

Execute

    lsblk

to list block devices (i.e., disks and partitions).
You should see something like nvme0n1 and/or nvme1n1.
For me, nvme1n1 should be Windows and nvme0n1 should be GNU/Linux.

To create the partitions on nvme0n1 (or recreate them), execute

    cfdisk /dev/nvme0n1

If partitions already exist, highlight the partitions and delete them.
Then select Write and enter yes to confirm.
Next, select New.
My motherboard should be UEFI only, so we need to create an EFI partition.
To create the EFI partition, delete the number currently listed (should be the drive's total space), replace it with 1G (1 GB), and hit enter.
This needs to be EFI, so select Type and then EFI System.
For the second partition, move down to Free space and select New.
For btrfs, keep the currently listed number (the remaining drive space) and hit enter.
Make sure its type is listed as Linux filesystem.
Now select Write and enter yes to write the changes to the disk.
Finally, select quit to finish with cfdisk.

The EFI partition needs to be the FAT32 filesystem type.
To make it so, execute

    mkfs.fat -F32 /dev/nvme0n1p1

(nmve0n1p1 should be the EFI partition, which you can confirm by its size).
For the remaining partition, execute

    mkfs.btrfs /dev/nvme0n1p2

To create the desired subvolumes, temporarily mount the btrfs partition:

    mount /dev/nvme0n1p2 /mnt

and create the subvolumes:

    btrfs subvolume create /mnt/@
    btrfs subvolume create /mnt/@home
    btrfs subvolume create /mnt/@snapshots

Now unmount the temporary mount:

    umount /mnt

------ Mounts and Base Install -----------------------------------------------------------

Next we need to mount the root partition:

    mount -o subvol=@ /dev/nvme0n1p2 /mnt

Then we create the following mount points:

    mkdir -p /mnt/home
    mkdir -p /mnt/.snapshots
    mkdir -p /mnt/boot/efi

Now mount the remaining subolumes and the boot partition:

    mount -o subvol=@home /dev/nvme0n1p2 /mnt/home
    mount -o subvol=@snapshots /dev/nvme0n1p2 /mnt/.snapshots
    mount /dev/nvme0n1p1 /mnt/boot/efi

Finally, we'll create a swapfile:

    mkdir /mnt/swap

For btrfs, we need to disable copy-on-write via

    chattr +C /mnt/swap

Next,

    dd if=/dev/zero of=/mnt/swap/swapfile bs=1G count=16 status=progress
    chmod 600 /mnt/swap/swapfile
    mkswap /mnt/swap/swapfile
    swapon /mnt/swap/swapfile

Here we can install the base system:

    basestrap /mnt base base-devel runit elogind-runit linux linux-firmware vim amd-ucode

(use intel-ucode for intel processors).
This will take some time to download and install.

Next we need to generate the filesystem table where the mount points are stored:

    fstabgen -U /mnt >> /mnt/etc/fstab

With Vim, you should be able to view the fstab file:

    view /mnt/etc/fstab

Exit view with

    :q

and hit enter.

=== Base Configuration ===========================================================================================================
==================================================================================================================================

Now we can enter into our system from the ISO via

    artix-chroot /mnt

Next we'll set the time zone.
Execute

    ls /usr/share/zoneinfo

to find your region, and then execute 

    ls /usr/share/zoneinfo/<region>

to find your city.
Then execute

    ln -sf /usr/share/zoneinfo/<region>/<city> /etc/localtime

and synchronize your hardware clock with your system clock:

    hwclock --systohc
